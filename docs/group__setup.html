◆ Modbus()

Modbus::Modbus ( uint8_t  u8id,
Stream &  port,
uint8_t  u8txenpin = 0 
)

Constructor for a Master/Slave.

For hardware serial through USB/RS232C/RS485 set port to Serial, Serial1, Serial2, or Serial3. (Numbered hardware serial ports are only available on some boards.)

For software serial through RS232C/RS485 set port to a SoftwareSerial object that you have already constructed.

ModbusRtu needs a pin for flow control only for RS485 mode. Pins 0 and 1 cannot be used.

First call begin() on your serial port, and then start up ModbusRtu by calling start(). You can choose the line speed and other port parameters by passing the appropriate values to the port's begin() function.

Parameters
u8idnode address 0=master, 1..247=slave
portserial port used
u8txenpinpin for txen RS-485 (=0 means USB/RS232C mode)

void Modbus::setTimeOut ( uint16_t  u16timeOut)

write communication watch-dog timer

Initialize time-out parameter.

Call once class has been instantiated, typically within setup(). The time-out timer is reset each time that there is a successful communication between Master and Slave. It works for both.
